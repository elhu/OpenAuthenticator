196   # Return an IO stream for the zipped entry.
 
197   #
 
198   # NOTE:  Originally this method used two approaches, Return a GZipReader
 
199   # directly, or read the GZipReader into a string and return a StringIO on
 
200   # the string.  The string IO approach was used for versions of ZLib before
 
201   # 1.2.1 to avoid buffer errors on windows machines.  Then we found that
 
202   # errors happened with 1.2.1 as well, so we changed the condition.  Then
 
203   # we discovered errors occurred with versions as late as 1.2.3.  At this
 
204   # point (after some benchmarking to show we weren't seriously crippling
 
205   # the unpacking speed) we threw our hands in the air and declared that
 
206   # this method would use the String IO approach on all platforms at all
 
207   # times.  And that's the way it is.
